## What was built (model-built)
The first on-screen HUD, themed to `design-handoff/gizmo-hud.png` (brass frames, cyan
HP, violet spark gem), scoped to v1 data only:
- `scenes/hud.tscn` (root `CanvasLayer` "Hud") + `scenes/hud_theme.tres` (brass
  `StyleBoxFlat` family). A `Control` tree anchored into the corners:
  Nameplate (GIZMO + cyan HP bar + "hp / max_hp"), LevelCluster (violet gem badge +
  XP bar), SparksReadout (gem + xp number + "SPARKS"), TimerPanel (M:SS run clock),
  LevelUpFlash (centered, hidden). Touched widgets use unique names (`%HpBar`…).
- `scripts/hud.gd` (`class_name Hud extends CanvasLayer`): `render(sim)` — a flat
  state→widget mapping (no game logic in the HUD); `static format_clock(seconds)`
  (M:SS, clamped); `_play_level_up_flash()` — a Tween fading the flash, driven by a
  `levelup` entry in `sim.last_events` (the 0009 seam, finally consumed).
- `scripts/game_controller.gd`: `@export var hud: Hud` and a guarded
  `if hud != null: hud.render(sim)` after the world sync. Optional — the bridge runs
  fine with no HUD assigned.
- `tests/run_hud_tests.gd`: a dedicated runner for the HUD's one pure function,
  `format_clock` (6 checks). Kept separate from `run_simulation_tests.gd` (which the
  user is editing in parallel) so the streams don't collide.

## Verified (implementation)
- `godot --headless --path godot --script res://tests/run_hud_tests.gd` →
  **PASS — 6 checks**, exit 0.
- godot-runtime MCP `validate`: `game_controller.gd`, `hud.gd`, `run_hud_tests.gd`,
  `hud.tscn` all valid (the `Hud` type resolves in project context).
- Live (MCP, `hud.tscn` standalone): the brass HUD renders — GIZMO nameplate + cyan
  HP "7 / 7", violet "1" level gem + XP bar, "0 SPARKS", "4:00" timer, flash hidden;
  in-run debug output had no errors. `render(sim)` exercised: HP→"4 / 7"/57%, level→2,
  Sparks→153, levelup → flash shown. Full in-game look pends the one-step integration.

## What the slice covers (to confirm with the learner)
The HUD as a CanvasLayer + anchored Control tree; theming via a `StyleBoxFlat` `Theme`;
the one-way `render(sim)` seam (sim is truth, HUD only reads — ADR 0002); unique-name
widget wiring; and the `last_events` → level-up flash (the 0009 seam paying off).

## Integration (the learner's one editor step)
Instance `scenes/hud.tscn` under `Main`, assign it to `GameController.hud` in the
Inspector. Left for the learner because `main.tscn` is in active art-pipeline flux;
the controller guards `hud == null` so nothing breaks until it's wired.
