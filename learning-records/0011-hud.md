## Review follow-ups (Codex review of 58fa582, 2026-06-20)
Applied (all in my files ‚Äî `hud.gd`, `hud.tscn`, `run_hud_tests.gd`, lesson, this record):
- **Count-down clock rounds up.** `format_clock` used `int(maxf(...))` (floor), so 0.9s
  left read `0:00` mid-run. Now `0 if seconds <= 0.0 else int(ceilf(seconds))` + two tests
  (0.1‚Üí`0:01`, 59.1‚Üí`1:00`). 8 checks.
- **Typed `render` seam.** `func render(sim: Simulation)` ‚Äî the controller‚ÜîHUD contract is
  now explicit, not duck-typed.
- **Tween ease order.** `set_trans`/`set_ease` now precede `tween_property` so the flash
  actually eases (Godot applies ease to tweeners appended *after* it).
- **Satisfying XP fill (user request).** The XP bar was a thin sliver and snapped. Now it's
  a wider bar (168√ó18) in the theme's recessed trough, and `_update_xp_fill` eases it toward
  each new fill (CUBIC/EASE_OUT, snaps on the first frame). HP still snaps (damage reads
  instantly); only the reward bar gets the flourish.
