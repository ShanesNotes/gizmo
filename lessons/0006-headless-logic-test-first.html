Gizmo · Lesson 0006 · The logic

Headless logic, test-first

Leave the scene behind for a moment: port the first slice of real game rules — and prove it with a test before it exists.

Why this, now

The player feels good, but there's no game yet — no progression, enemies, or stakes. Those rules already exist, battle-tested, in game-src-phaser/src/game/simulation.ts (our source of truth). We'll port them into Godot one slice at a time. And because this logic is pure math — no models, no camera — we can develop it headless and test-first: write a failing test, then the code to pass it. That's how we port 1,700 lines of rules without it turning into guesswork.

The idea: red → green

Test-Driven Development is a tight loop:

  1. Red — write a test that states what the code should do. Run it; it fails (the code doesn't exist yet).
  2. Green — write the smallest code that makes the test pass. Run it; it passes.
  3. Refactor — clean up, with the test as your safety net.

Writing the test first forces you to decide what "correct" means before you're tangled in implementation — and it gives you a permanent check that survives every future change.

Our first slice: XP & leveling

From simulation.ts we're porting two things: the curve nextXpForLevel(level) (how much XP each level needs) and the level-up rule (gaining XP past the threshold bumps your level and carries the remainder). Small, pure, foundational — a perfect first port.

Step 1 (RED) — write the test runner first

We don't need a heavy test framework. Create tests/run_simulation_tests.gd — a tiny runner that's just a script Godot executes headlessly:

extends SceneTree

# Tiny dependency-free test runner. Run with:
#   godot --headless --path godot --script res://tests/run_simulation_tests.gd
# Exits 0 if all pass, 1 if any fail.

var _passed := 0
var _failed := 0

# Load by path so this runner works headless without an editor import step
# (the global class_name cache isn't refreshed by a raw --script run).
const Sim := preload("res://scripts/simulation.gd")

func _initialize() -> void:
	print("Running simulation tests…")
	_test_next_xp_curve()
	_test_xp_below_threshold()
	_test_level_up_carries_remainder()
	print("")
	if _failed == 0:
		print("PASS — %d checks" % _passed)
		quit(0)
	else:
		printerr("FAIL — %d passed, %d failed" % [_passed, _failed])
		quit(1)

func _check(desc: String, condition: bool) -> void:
	if condition:
		_passed += 1
		print("  ok  - ", desc)
	else:
		_failed += 1
		printerr("  FAIL - ", desc)

func _test_next_xp_curve() -> void:
	_check("level 1 needs 92 xp", Sim.next_xp_for_level(1) == 92)
	_check("threshold grows with level", Sim.next_xp_for_level(2) > Sim.next_xp_for_level(1))

func _test_xp_below_threshold() -> void:
	var sim := Sim.new()
	var leveled := sim.add_xp(10)
	_check("a little xp does not level you", not leveled and sim.level == 1 and sim.xp == 10)

func _test_level_up_carries_remainder() -> void:
	var sim := Sim.new()
	var leveled := sim.add_xp(100)  # level-1 threshold is 92
	_check("crossing the threshold levels up", leveled and sim.level == 2)
	_check("remainder carries over (100 - 92 = 8)", sim.xp == 8)
	_check("next threshold becomes level 2's", sim.next_xp == Sim.next_xp_for_level(2))

Run it now — from the repo root, in a terminal:

godot --headless --path godot --script res://tests/run_simulation_tests.gd

It fails, loudly: Parse Error: Identifier "Simulation"… — the runner can't even load because simulation.gd doesn't exist. That's a perfectly good Red: the test demands something that isn't there yet.

Step 2 (GREEN) — write the code to pass

Create scripts/simulation.gd — the headless logic module, ported faithfully from the TypeScript:

class_name Simulation
extends RefCounted

## Headless game logic, ported from game-src-phaser/src/game/simulation.ts.
## First slice: player XP and leveling. No scene, no rendering — pure and testable.

var level: int = 1
var xp: int = 0
var next_xp: int = 0

func _init() -> void:
	next_xp = next_xp_for_level(level)

## XP needed to advance FROM `lvl` to the next level.
## Faithful to nextXpForLevel() in simulation.ts.
static func next_xp_for_level(lvl: int) -> int:
	var base := 56.0 + lvl * 14.0 + pow(lvl, 1.8) * 12.5
	var pacing := 1.12 if lvl == 1 else 1.48 + minf(0.36, (lvl - 2) * 0.07)
	return int(floor(base * pacing))

## Grant XP. Returns true if it triggered a level-up.
## Mirrors the level-up block in simulation.ts: the remainder carries over.
func add_xp(amount: int) -> bool:
	xp += amount
	if xp >= next_xp:
		xp -= next_xp
		level += 1
		next_xp = next_xp_for_level(level)
		return true
	return false

Run the same command again. Green:

Running simulation tests…
  ok  - level 1 needs 92 xp
  ok  - threshold grows with level
  ok  - a little xp does not level you
  ok  - crossing the threshold levels up
  ok  - remainder carries over (100 - 92 = 8)
  ok  - next threshold becomes level 2's

PASS — 6 checks

Why it's shaped this way

  • extends RefCounted — no Node, no scene. This is pure logic; it can run, and be tested, with no game window. That's the "headless" win.
  • Faithful to the source. The formula and the carry-the-remainder rule are copied from simulation.ts, not reinvented. When in doubt, the TypeScript wins.
  • One behavior per check. Each _check tests one fact, so a failure points at exactly what broke.
