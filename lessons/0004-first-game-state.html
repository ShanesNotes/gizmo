Your First Game State

Leave the editor's visuals for a moment and write the data the whole game runs on — a function that returns Gizmo's opening state. Pure logic, no rendering.

0003 ✓ → 0004 · first game state → 0005 · first headless test

The one idea

A game is just data that changes over time. Before anything moves on screen, you need a clear snapshot of the world at t = 0: where the player is, how big the world is, how long a run lasts, what's been collected. In Gizmo we keep that snapshot in a Dictionary — a bag of named values — built by one function, create_game_state().

This is headless work: a script with no node, no scene, no pixels. That's deliberate — pure rules are the easiest thing to get right and to test (which is exactly Lesson 0005).

Faithful, but small on purpose. The finished seed tracks ~60 fields (every economy, timer, and streak). We start with the core handful and grow it lesson by lesson — same shape, same values, just less of it. Source of truth: game-src-phaser/src/game/simulation.ts.

Do it

  1. In the FileSystem dock (bottom-left), right-click the scripts/ folder → Create New → Script…
  2. Set Inherits to RefCounted, Path to res://scripts/simulation.gd, and click Create. (No node needed — this is pure data.)
  3. Replace the file with this, then save (Ctrl+S). Use the Copy button — it pastes with the tabs GDScript needs: 
    class_name Simulation
extends RefCounted

const WORLD_WIDTH: int = 2600
const WORLD_HEIGHT: int = 1700
const RUN_DURATION: float = 240.0
const START_X: float = WORLD_WIDTH / 2.0
const START_Y: float = WORLD_HEIGHT / 2.0

func create_game_state() -> Dictionary:
    return {
        "phase": "playing",
        "elapsed": 0.0,
        "run_duration": RUN_DURATION,
        "world": {
            "width": WORLD_WIDTH,
            "height": WORLD_HEIGHT,
        },
        "player": {
            "x": START_X,
            "y": START_Y,
            "level": 1,
            "xp": 0,
        },
        "enemies": [],
        "pickups": [],
    }
  4. Peek at it. Open scripts/title.gd and, inside _ready(), add these two temporary lines, then press F5: 
    var sim := Simulation.new()
print(sim.create_game_state())
    The Output panel prints the whole state dictionary. That's your game's starting world. Delete the two lines afterward — Lesson 0005 gives us a real test instead of temporary prints.

What each piece does

class_name SimulationRegisters this class engine-wide, so any script can write Simulation.new(). extends RefCountedA lightweight object with no node/scene overhead — perfect for pure logic. const WORLD_WIDTH: intA fixed, typed value. : int / : float catch type mistakes early. func create_game_state()Builds and returns the opening snapshot — never mutates anything outside. { "key": value }A Dictionary: named fields, nestable (world and player are dicts inside the dict).
Why RefCounted and not a Node? Nodes live in scenes and cost render/processing overhead. The simulation is pure rules — it should run headless, in a test, with no scene at all. RefCounted frees itself automatically when nothing references it. Godot docs ↗
That's the win. You wrote the data backbone of Gizmo — one function that defines the world at the start of a run. Everything from here (movement, pickups, leveling) is just changing this dictionary over time.

Prove it

You saw it print. For a no-editor check that the script is valid, from the repo root:

godot --headless --path godot --check-only --script res://scripts/simulation.gd

No errors and exit 0 = the class parses. Next lesson we assert the values are right automatically.

Mental model to keep

  • A game = data over time. Start with a clear t = 0 snapshot.
  • Dictionary = named, nestable values. Gizmo's whole state lives in one.
  • RefCounted for pure logic — no node, runs headless, easy to test.
  • Type your constants (: int, : float) — small habit, fewer bugs.
Stuck or curious? Ask your teacher: "what are all 60 fields the seed tracks?", "why a Dictionary instead of a custom class?", or "what does START_X = WORLD_WIDTH / 2.0 evaluate to?" (it's 1300.0).
