Gizmo · Lesson 0013 · Balance

Balance: make the run lethal

0012 closed the loop, but you could only win or die by pressing F8/F9. 0013 tunes the run so it's naturally survivable and lethal — and, just as importantly, builds a deterministic test harness that pins that curve so a future tweak can't silently break it.

Why this, now — and how (metric-driven)

A balance pass is the difference between "the loop runs" and "the loop is a game." But balance by vibes regresses the moment you touch a constant. So the rule for 0013 was: define the target curve from reference/game-balance-reference.md first, then tune to hit it, then pin it with tests. The reference is the north star: TTK bands (§5.4), the horde clear-vs-spawn pressure equation (§5.3), one-shot control (§3.4), spawn failure modes (§6.2).

Part 1 — enemy roles (rules-only, still no waves)

One enemy type can't make a curve. 0013 ports the source roles (simulation.ts:259) as rules-only variants — same chaser logic, different stats — gated by the clock, not by discrete rounds (ADR 0003):

nibbler  — baseline trash         (from 0s)
dasher   — fast trash             (unlocks 30s)   §5.4 trash: die fast, punish sloppy kiting
brute    — bruiser, 4-shot HP     (unlocks 66s)   §5.4 bruiser: force target priority
warden   — mid priority target    (unlocks 98s)

The director still spends a time-ramped budget (0010), but now it picks a role and saves budget for a pricier one instead of always defaulting to nibblers (mirrors simulation.ts:684). The mix is a deterministic cadence over the clock — continuous pressure with variety, never a "WAVE x/5" spike.

Part 2 — survivable AND lethal

The §5.3 horde equation says: a run is fair when ClearPressure ≥ SpawnPressure at intended build power. Tuned for v1:

  • More early threat so standing still actually kills you: BUDGET_BASE 0.45 → 0.9, BUDGET_PRESSURE_GAIN 9.5 → 10.5.
  • Contact knockback (1.35 m) — ports the source bump recoil so one touch isn't an instant death-spiral (§3.4 one-shot control). It only fires on a real hit (after i-frames), so it can't gift free escapes.
  • Reachable XP: PICKUP_RADIUS 1.8 → 2.4 (§6.2 "XP inaccessible" failure mode).
  • Level-ups that matter: a prototype Spark Chain (the canon weapon, simulation.ts:267) auto-scales with level — more chain targets, a shorter cooldown (floored), more damage every 3 levels. This is a temporary stand-in so leveling has teeth before the real Core Matrix / upgrade-choice system exists (ADR 0001); the combat seam is unchanged, so those lessons can swap it out.

Part 3 — pin the curve (the honest part)

Tests that pass aren't the same as tests that verify. The first cut of this harness passed but several checks were tautological or geometry-bound — they'd green-light a real balance regression. So the harness was hardened into a genuine deterministic regression guard (run_balance_tests.gd, PASS — 32):

TTK bands (§5.4)      trash ≈0.55s · brute ≈2.05s · warden ≈1.55s   (tight, role-separating)
one-shot guard (§3.4) max_hit_delta ≤ 1 in every profile            (no contact ever spikes >1 HP)
stationary            loses ~55s, first damage ~46s                 (lethal, but no cheap early death)
mistake-kite          loses ~187s under real board pressure          (weak routing eventually fails)
decent kite           WINS with HP left                              (a fair "decent play" proxy — below)
60s pressure probe    spawns/kills/max-alive/dashers in tuning bands (moves if you retune BUDGET_*)

The decent-kite profile is the one that earns the word "decent": it moves at Gizmo's true 6.0 m/s, only sees enemies within 5.5 m, and re-decides every 0.2 s (straight lines between), bouncing off the arena walls. It is not an omniscient frame-perfect avoider — so "it wins" honestly means a competent human can win, not that only a robot can. The 60s probe is the regression-sensitive one: its spawn/kill bands move when you touch BUDGET_*, so a careless ±15% retune trips a red test.

Prototype balance — verified by sim proxies, NOT "final game balance"

Be precise about what's proven: the curve is pinned by deterministic sim profiles, which catch structural and gross-tuning regressions. They are not a human playtest. Still open: a real hands-on feel pass, and enemy behaviour around the world-kit pillars — the Simulation is a flat-plane positional model with enemy separation but no knowledge of 3D obstacles, so clipping/stacking can only be judged live. And the Spark Chain auto-scaling is a balance bridge, not the final upgrade design.

Done means…

Done = a run is naturally survivable and lethal without F8/F9; the curve is grounded in the balance reference; and run_balance_tests.gd is green (32 checks) as a regression guard, with the full suite still passing (sim 69 · hud 8 · end 6 · controller 10). You can explain why "tests pass" needed sharpening into "tests verify," why the roles are continuous pressure (not waves), and why this is a prototype balance — proven by sim proxies, awaiting a live playtest.

Deferred — named, not built

The real Core Matrix / upgrade-choice system (replaces the Spark Chain auto-scale), the live human playtest + world-kit collision check, elite/boss tiers (§5.4 has the bands ready), and finer second-pass tuning once those exist.
