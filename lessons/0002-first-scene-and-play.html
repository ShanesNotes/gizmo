Your First Scene — and Press Play

Build a scene, drop in one node you can actually see, and run the game for real. This is the first time Gizmo moves.

0001 · create the project ✓ → 0002 · first scene + Play → 0003 · first GDScript

The one idea

A Godot game is a tree of nodes, saved as a scene — a .tscn file. A node is one small building block (a camera sees, a mesh draws, a label shows text). You stack and nest them into a tree, and that tree is your game screen.

Gizmo's rules stay flat, but its screen is an orthographic 2.5D stage: a Node3D world viewed by a non-perspective Camera3D. When you press Play, Godot loads one chosen scene — the main scene — and runs it in a real window.

Main — Node3D (the 2.5D stage root)
  ├─ World — Node3D
  │  └─ StageMarker — MeshInstance3D
  ├─ CameraRig — Node3D
  │  └─ Camera3D — orthographic
  ├─ KeyLight — DirectionalLight3D
  └─ HUD — CanvasLayer → Label

Do it

  1. Create the root node. In the Scene dock, click 3D Scene. Rename the new Node3D to Main. This is the stage root.
  2. Add the quiet stage. Add child Node3D named World. Under it, add MeshInstance3D named StageMarker. In the Inspector set its Mesh to a small BoxMesh or PlaneMesh. Keep it simple: this is a marker that proves the camera can see the stage.
  3. Add the camera. Add child Node3D named CameraRig, then add Camera3D under it. Set Projection to Orthogonal, Size to about 14.4, move it up/back, and rotate it down until it sees the marker.
  4. Add light. Add a DirectionalLight3D named KeyLight. Rotate it a bit so the marker is readable.
  5. Add a screen-space title. Add CanvasLayer named HUD, then a Control named Root, then a Label named Title. Set its text to Gizmo. HUD nodes live on the screen; they do not move with the 3D stage.
  6. Save the scene. Press Ctrl+S. Save it inside the scenes/ folder with the filename main.tscn — filenames are snake_case. Its project path is now res://scenes/main.tscn.
  7. Press Play (F5, or the ▶ top-right). Godot says "no main scene defined" and offers to set one — click Select Current. A window opens showing your tiny stage plus Gizmo. That's your game running.
Why the "Select Current" prompt? Pressing Play needs to know which scene to launch. Choosing Select Current records this scene as the project's main scene — Godot writes run/main_scene="res://scenes/main.tscn" into project.godot. You only answer this once.
That's the win. A real OS window opened and ran a scene you built — stage root, camera, light, HUD, saved, launched. You now know the whole spine of Godot: nodes → scene → main scene → Play. Every screen in Gizmo is this same move, scaled up.

Prove it

On screen you already saw it run. For a second confirmation, from a terminal at the repo root:

grep main_scene godot/project.godot
ls godot/scenes/main.tscn

The first prints run/main_scene="res://scenes/main.tscn"; the second shows the file exists. Want a no-window check? This loads the main scene headlessly and quits clean: 

godot --headless --path godot --quit-after 2

Mental model to keep

  • Scene = a tree of nodes saved as a .tscn file. Games are scenes.
  • Node = one job. A Camera3D sees, a MeshInstance3D draws, a Label shows text. You compose them.
  • Main scene = what Play loads first. Set once, lives in project.godot.
  • Names: PascalCase nodes, snake_case files. Main the node, main.tscn the file.
You just built the real skeleton. The finished reference port opens with exactly this: a Main (Node3D) root, orthographic Camera3D, quiet stage marker, and HUD label. You're not on a detour — this is the first stage of the actual game, and the next lessons grow Gizmo's logic onto it.
