State on Screen

Connect the headless simulation to the scene: Main owns a Simulation, ticks it every frame, and shows elapsed climbing live. The data finally surfaces.

0006 ✓ → 0007 · state on screen → 0008 · the player moves

The one idea

Two worlds have been separate: the simulation (pure data, tested headless) and the scene (nodes you see). 0007 joins them. The Main node holds a Simulation instance and its state dictionary, calls update_state once per frame using the frame's delta, and writes a value from state onto a Label. That's the shape of every Godot game: a script owns the model and renders it each frame.

Clean split, on purpose. The simulation never touches a node; the scene never computes rules. The scene just reads state and draws it. That separation is exactly why the sim stays testable — and it's how the seed is organized too.

Do it

  1. Retire the sway. Select the Title node → right-click → Detach Script (the title.gd from 0003 was scaffolding; you can delete that file from the FileSystem dock).
  2. Give Main a brain. Select the Main node → Attach Script → path res://scripts/main.gd, Inherits Node3D → Create.
  3. Replace main.gd with this and save: 
    extends Node3D

var sim := Simulation.new()
var state: Dictionary = {}

@onready var title: Label = %Title

func _ready() -> void:
    state = sim.create_game_state()

func _process(delta: float) -> void:
    sim.update_state(state, {}, delta)
    title.text = "Elapsed: %.1f s" % state["elapsed"]
  4. Press Play (F5). The label now reads Elapsed: 0.3 s, 0.6 s… climbing in real time, driven by the very update_state your tests check.

What each piece does

var sim := Simulation.new()The scene owns one simulation instance for the whole run. @onready var title := %TitleGrabs the HUD Label once the node tree is ready. _ready(): state = create_game_state()Build the opening state when the scene starts. _process(delta): update_state(...)Advance the sim by the real frame time. delta becomes dt — clamped safely inside. title.text = "%.1f" % …Render one field of state to the screen, every frame.
Notice what you didn't do: no game logic went into main.gd. It only builds state, ticks it, and draws it. All the rules still live in the tested simulation.gd. Keep that wall up — it's what keeps Gizmo provable.
That's the win. The headless model and the visible scene are now one running game. From here, "show it on screen" is always this move: read a field of state, draw it.

Prove it

The timer climbs on screen — that's the proof. And your headless test still passes untouched:

godot --headless --path godot --script res://tests/run_simulation_tests.gd

The sim didn't change, so it's still green. Visual and headless agree.

Mental model to keep

  • Script owns the model, renders each frame. Build in _ready, advance + draw in _process.
  • @onready + $Name = a safe reference to a child node.
  • The sim stays pure. Logic in simulation.gd; the scene only reads and draws.
Stuck or curious? Ask: "why @onready instead of just $Title at the top?", or "what happens if I show state['phase'] too?" Next: 0008 — input arrives, and the player actually moves around a scrolling world.
