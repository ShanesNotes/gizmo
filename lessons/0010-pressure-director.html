Gizmo · Lesson 0010 · Escalating pressure

Pressure director: the clock is the boss

Enemies stop trickling in on a fixed timer — a pressure curve ramps the spawn rate as the run pressure rises. The snowball begins.

Why this, now

0009 closed the loop — kill → Spark → XP → level. But the spawn was a flat trickle (one nibbler every 1.2s, forever), so the run never built. A rogue-lite needs the pressure to rise: in a horde game the clock is the boss (balance §5.2). 0010 replaces that seed cadence with a director — a pressure curve off the run clock fills a spawn budget that's spent on enemies, so spawns accelerate over the run. Tuning intent: reference/game-balance-reference.md §5.2/§5.3 (scaling, horde tuning), §6.1 (spawn budget); mechanics: simulation.ts (updateDirector, heatCurve).

Not waves — director pressure (ADR 0003)

This is not a "WAVE 1/5" round structure (that lived only in old concept art). Per docs/adr/0003-director-pressure-not-discrete-waves.md, v1 is a continuous, director-driven pressure curve: enemies spawn faster as the run goes on, with no player-facing wave counter. Tougher enemy kinds and special threats are real later milestones — but introduced deliberately in their own lessons, not because a wave number implies them.

Part 1 — the pressure curve (logic, test-first)

One number drives this slice: pressure(), a 0..1 time-pressure scalar that rises over the run. It mirrors the time term of simulation.ts's heatCurve (1727-1730) — an eased ramp (exponent 2.15), so the opening is calm and the late game accelerates:

func pressure() -> float:
	var t := (clampf(elapsed / run_duration, 0.0, 1.0)) if run_duration > 0.0 else 1.0
	return clampf(1.0 - pow(1.0 - t, PRESSURE_EASE), 0.0, PRESSURE_MAX)

At the start pressure ≈ 0; halfway it's already ~0.78 (the easing front-loads difficulty); by the run's end it's 1.0. The source heatCurve can later reach 1.42 after + level*0.014 + kills*0.00135; those refinements are deferred until level/kills are part of the active director.

Test-first — pressure ramps 
func _test_pressure_curve_ramps() -> void:
	var sim := Sim.new()
	sim.elapsed = 0.0
	_check("pressure starts at zero", absf(sim.pressure()) < 0.001)
	sim.elapsed = sim.run_duration * 0.25
	var quarter := sim.pressure()
	sim.elapsed = sim.run_duration * 0.75
	var late := sim.pressure()
	_check("pressure rises as the run progresses", late > quarter and quarter > 0.0)
	sim.elapsed = sim.run_duration
	_check("time-only pressure reaches 1.0 by the run's end", absf(sim.pressure() - 1.0) < 0.001)

Part 2 — the budget director (the ramp)

The old fixed-timer spawn is gone. In its place, a budget fills each tick at a rate that climbs with pressure, and we spend it on enemies (one nibbler costs NIBBLER_COST, 1.1) up to the alive cap — a faithful core of updateDirector (simulation.ts:666-689). A per-tick batch limit keeps a huge late-game budget from stalling one frame:

if spawn_enabled:
	var budget_rate := BUDGET_BASE + pow(pressure(), BUDGET_PRESSURE_EXP) * BUDGET_PRESSURE_GAIN
	_spawn_budget += budget_rate * dt
	var spawned := 0
	while _spawn_budget >= NIBBLER_COST and enemies.size() < max_enemies and spawned < MAX_SPAWNS_PER_TICK:
		_spawn_nibbler(gizmo_position)
		_spawn_budget -= NIBBLER_COST
		spawned += 1

At pressure 0 the budget fills at 0.45/s — the first nibbler takes ~2.4s, a calm open. By 180s in, pressure ≈ 0.96 and the rate is ~9/s — a steady swarm. The spawn_enabled flag lets unit tests turn the director off and place enemies by hand (that's the refactor behind 0008's tests now).

Test-first — the ramp, the cap, the off-switch 
func _test_director_ramps_spawn_rate() -> void:
	# Same one-second window, low pressure vs high pressure -> more spawns under pressure.
	var early := Sim.new()
	early.attack_cooldown = 9999.0   # count spawns, don't let the weapon thin them
	early.elapsed = 0.0
	for i in 20:
		early.tick(0.05, Vector3.ZERO)
	var late := Sim.new()
	late.attack_cooldown = 9999.0
	late.elapsed = 180.0             # pressure ~0.96
	for i in 20:
		late.tick(0.05, Vector3.ZERO)
	_check("the director spawns faster under high pressure", late.enemies.size() > early.enemies.size())
	_check("high pressure produces a real burst", late.enemies.size() >= 3)

func _test_director_respects_cap() -> void:
	var sim := Sim.new()
	sim.attack_cooldown = 9999.0
	sim.hp = 9999                    # contact must not end the run mid-test
	sim.max_enemies = 5
	sim.elapsed = 200.0              # very high pressure -> wants far more than 5
	for i in 60:
		sim.tick(0.05, Vector3.ZERO)
	_check_eq("the director never exceeds the enemy cap", sim.enemies.size(), 5)

Deferred — named hooks (ADR 0003)

The real updateDirector does more, all deferred to their own lessons so this slice stays one win: the player-power budget terms (pScore, late/power pressure); enemy intensity scaling (HP/speed/XP rising with pressure, simulation.ts:1088-1116); multiple kinds unlocking over time (dasher @30s, brute @66s, warden @98s, ENEMY_SPECS 258-263); and deliberately designed special encounters. Naming them now keeps the architecture honest without building ahead of need.
