Gizmo · Lesson 0012 · Win / lose

Win / lose: close the loop

The run has been able to end since 0007 — the timer runs out, or Gizmo's HP hits zero — but nothing showed for it. 0012 surfaces those two endings as a brass screen with your run stats and a RETRY button. That closes the v1 loop: play → win or die → go again.

Why this, now

The Simulation already reaches two end states: PHASE_COMPLETE when the clock runs out (a win — survive the run, simulation.ts:490) and PHASE_GAMEOVER when HP reaches 0 (a loss, simulation.ts:737). Until now the run just froze there. 0012 reads that phase and shows a result. The architecture holds (ADR 0002): the Simulation owns the phase, the screen only renders it.

Part 1 — the outcome is pure (test-first)

What text does each ending get? That's a pure function of the phase — no nodes, no scene — so it's a static that gets a test before any UI exists:

# end_screen.gd
static func outcome(phase: String) -> Dictionary:
	if phase == Simulation.PHASE_COMPLETE:
		return {"title": "RUN COMPLETE", "flavor": "Gizmo survived the run.", "win": true}
	if phase == Simulation.PHASE_GAMEOVER:
		return {"title": "GIZMO OFFLINE", "flavor": "Gizmo's chassis gave out.", "win": false}
	return {"title": "", "flavor": "", "win": false}   # still playing → no banner
func _test_outcome() -> void:
	var win := EndScreen.outcome(Simulation.PHASE_COMPLETE)
	_check_eq("complete -> win", win["win"], true)
	_check_eq("complete -> RUN COMPLETE", win["title"], "RUN COMPLETE")
	var loss := EndScreen.outcome(Simulation.PHASE_GAMEOVER)
	_check_eq("gameover -> not win", loss["win"], false)
	_check_eq("gameover -> GIZMO OFFLINE", loss["title"], "GIZMO OFFLINE")
	var playing := EndScreen.outcome(Simulation.PHASE_PLAYING)
	_check_eq("playing -> no outcome (empty title)", playing["title"], "")   # defensive
	_check_eq("playing -> not win", playing["win"], false)

godot --headless --path godot --script res://tests/run_end_screen_tests.gd → PASS — 6 checks. The defensive "playing → empty" case means a mis-timed call shows nothing, never a wrong banner.

Part 2 — the end screen (brass overlay)

The screen is its own CanvasLayer on a higher layer than the HUD (3 vs 2), so it sits on top. It reuses the same scenes/hud_theme.tres the HUD uses, so it's the same brass family for free. A dim ColorRect darkens the frozen game behind a centered panel:

EndScreen (CanvasLayer, layer 3)        scripts/end_screen.gd
└─ Root (Control, Full Rect)            theme = hud_theme.tres
   ├─ Dim    (ColorRect, full)          → darkens the world behind
   └─ Center (CenterContainer)
      └─ Panel (PanelContainer, brass)
         └─ VBox
            ├─ TitleLabel   → "RUN COMPLETE" / "GIZMO OFFLINE"
            ├─ FlavorLabel  → one line of flavor
            ├─ Stats (Grid) → SURVIVED 4:00 · LEVEL 5 · SPARKS 120
            └─ RetryButton  → "RETRY"

Touched widgets are unique (%TitleLabel, %SurvivedValue…), and the script hides %Root in _ready — so the screen is invisible until the run actually ends. show_outcome(sim) fills it and reveals it:

# end_screen.gd
func show_outcome(sim: Simulation) -> void:
	var copy := outcome(sim.phase)
	_title.text = copy["title"]
	_flavor.text = copy["flavor"]
	_survived_value.text = Hud.format_clock(floorf(minf(sim.elapsed, sim.run_duration)))
	_level_value.text = str(sim.level)
	_sparks_value.text = str(sim.xp)
	_root.visible = true
	_retry_button.grab_focus()
One formatter, two rounding rules

The survived time reuses Hud.format_clock from 0011 — but the HUD clock counts down and rounds up (so any time left still shows), while "time survived" counts up and should round down (you died at 2:17.6, not 2:18). Same M:SS formatter; the caller picks the rounding by pre-flooring the value it passes in. The minf(…, run_duration) also clamps a win's tiny overshoot back to a clean 4:00.

Part 3 — show it once, on the transition

The controller already ticks the Simulation every frame. It just has to notice the one frame the run leaves playing and reveal the screen — exactly once, not every frame after:

# game_controller.gd
@export var end_screen: EndScreen          # assign an end_screen.tscn instance (optional)
var _prev_phase: String = Simulation.PHASE_PLAYING

func _physics_process(delta: float) -> void:
	...
	if hud != null:
		hud.render(sim)
	# the frame the run leaves "playing" (win or loss), show the end screen once
	if end_screen != null and sim.phase != Simulation.PHASE_PLAYING and _prev_phase == Simulation.PHASE_PLAYING:
		end_screen.show_outcome(sim)
	_prev_phase = sim.phase

Comparing this frame's phase against the previous one is the whole trick: it fires on the edge (playing → ended), so the panel and the button's focus aren't re-set 60 times a second.

RETRY = a fresh run

The button calls get_tree().reload_current_scene(). That tears the scene down and rebuilds it from disk — a brand-new Simulation.new(), timer back to 4:00, HP full, no enemies. No reset bookkeeping to maintain: reloading is the reset. (When a run later carries meta-progress, that's a save concern for its own lesson — ADR 0001.)

Wire it up (one editor step)

Both 0011's HUD and this screen are self-contained scenes, so the only integration is: instance scenes/hud.tscn and scenes/end_screen.tscn under Main, then drag each into the GameController's HUD and End Screen slots. (main.tscn is in active art-pipeline flux, so this last click is yours — the controller guards both against null, so nothing breaks until they're assigned.)

Faithful, not invented (ADR 0001)

The loss copy is "Gizmo's chassis gave out" — Gizmo's HP ran out, the robot was destroyed. It is deliberately not "the Spark faded": HP is not the Spark of Humanity (ADR 0001), and the Spark's real stakes arrive with that system. The flavor here stays factual so it doesn't quietly invent a mechanic we haven't built.

Done means…

Done = end_screen.tscn renders the brass result; outcome is green (6 checks); a finished run shows RUN COMPLETE (win) or GIZMO OFFLINE (loss) with SURVIVED / LEVEL / SPARKS; and RETRY starts a fresh run. That's verified standalone and via a runtime probe — the full in-game loop closes the moment you finish the one wiring step above. You can explain why the outcome is a pure static, why the controller fires on the phase edge, and why RETRY just reloads the scene. With this, v1's core loop is complete: play → win or die → go again.

Deferred — named, not built

The level-up choice screen (the Phaser "levelup" phase — pick an upgrade mid-run), richer end-screen narrative + the real Spark of Humanity stakes (ADR 0001), a score / best-run record, and a proper title/start screen — each lands with its system, after v1 ships.
