Your First GDScript

Attach code to a node and watch it run — a printed message, then a label that moves on its own. This is where Gizmo stops being still.

0001 ✓ → 0002 ✓ → 0003 · first GDScript → 0004 · first game state

The one idea

A node knows how to be a thing — a Label shows text, a Node3D holds our 2.5D stage. A script gives a node behavior: your own code that runs as the game plays. You attach one script to one node, and Godot runs it.

Two functions do almost everything. _ready() runs once, the moment the node wakes up — perfect for setup. _process(delta) runs every frame — that's your animation and game loop. delta is the seconds since the last frame; multiply motion by it so the game moves the same speed on any machine.

Why move a label? Gizmo is a rogue-lite — enemies swarm in escalating waves. Before we port real movement from the seed, you get the feel of "code drives motion" on something you already built. Same _process loop, just one node instead of a hundred.

Do it

  1. Select the Title node in the Scene dock (the Label you made in 0002).
  2. Attach a script. Click the Attach Script icon above the Scene tree (a scroll with a +), or right-click Title → Attach Script.
  3. In the dialog: Inherits is Label (leave it). Set the Path to res://scripts/title.gd — snake_case, in the scripts/ folder. Click Create. The Script editor opens with a small template.
  4. Replace the template with this, then save (Ctrl+S): 
    extends Label

var start_x := 0.0
var elapsed := 0.0

func _ready() -> void:
	print("Gizmo is alive!")
	start_x = position.x

func _process(delta: float) -> void:
	elapsed += delta
	position.x = start_x + sin(elapsed * 3.0) * 60.0
  5. Press Play (F5). The label gently sways left and right — and the Output panel (bottom of the editor) prints Gizmo is alive!.

What each line does

extends LabelThis script is a Label, with your extra behavior bolted on. var start_x := 0.0A variable on the node; := lets Godot infer the type (float). func _ready()Runs once when the node enters the scene. We print and remember the start position. func _process(delta)Runs every frame. delta = seconds since last frame. sin(elapsed * 3.0) * 60.0A smooth wave from −60 to +60, so the label sways around where you placed it.
Indentation matters in GDScript — it uses tabs, like Python uses spaces. When you press Enter after a line ending in :, Godot's editor indents for you. If you paste and see an "unexpected indent" error, select all and press Tab/Shift+Tab to fix it. Don't mix tabs and spaces.
That's the win. You wrote code, attached it to a node, and it ran — printing on startup and animating every frame. _ready + _process is the heartbeat of every Godot game, from this one label to Gizmo's whole swarm.

Prove it

You saw it sway and print. For a check without opening the editor, from the repo root:

godot --headless --path godot --check-only --script res://scripts/title.gd

No output and exit 0 means the script parses cleanly. Running the game headless for two seconds will also fire _ready() and print your message: 

godot --headless --path godot --quit-after 2

Mental model to keep

  • Script = a node's behavior. One script attaches to one node; extends says what kind of node it is.
  • _ready() runs once — setup. _process(delta) runs every frame — the loop.
  • Always scale motion by delta so speed is the same on a fast or slow machine.
  • GDScript indents with tabs. Let the editor do it; never mix tabs and spaces.
