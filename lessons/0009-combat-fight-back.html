Combat: fight back & close the loop

Auto-fire kills enemies, they drop Sparks, you vacuum them into levels — the dopamine circuit, complete.

Why this, now

0008 gave you a crowd that hurts. Now Gizmo fights back, and the run finally loops: auto-fire damages enemies → they die → they drop Sparks → you collect them → Sparks are XP, so you level up (the 0006 curve). Kill, gather, grow — that's the engine of every Vampire Survivors / Diablo / Megabonk run. Tuning intent: reference/game-balance-reference.md §2 (damage) & §5 (XP/progression); mechanics: simulation.ts (updateWeapons, dealDamage).

Auto-fire (your call) — and what's deferred

Per the design lineage, the weapon is auto-firing: no aim button, pure positioning. On a cooldown it hits the nearest enemy in range. The combat logic lives in Simulation (ADR 0002), reusing 0006's add_xp and 0008's enemies. Deferred (named later lessons): weapon VFX — tracers, projectiles, GPU particles (a particles/shader lesson), so for now the shot itself is invisible; the feedback is enemies vanishing and Sparks flying out. Also later: crits, multi-target, Cache/Heart drops, magnet upgrades, and waves/elites/bosses (0010+).

Part 1 — the combat loop (logic, test-first)

Two new data agents close the circuit. The enemy gains an xp_value, and a dropped Spark is its own tiny agent:

class Pickup extends RefCounted:
	var position := Vector3.ZERO
	var value: int = 0

tick gains two phases after the enemies update — the weapon fires, then Sparks are collected:

func _update_weapon(dt: float, gizmo_position: Vector3) -> void:
	_attack_timer += dt
	if _attack_timer < attack_cooldown:
		return
	_attack_timer -= attack_cooldown
	var target: Enemy = null
	var best := INF
	for e in enemies:
		if e.hp <= 0.0:
			continue  # a corpse is not a target (simulation.ts:1714 filters hp > 0)
		var d := gizmo_position.distance_to(e.position)
		# reachable when the enemy's BODY is in range (simulation.ts:1716: dist <= range + radius)
		if d <= attack_range + e.radius and d < best:
			best = d
			target = e
	if target == null:
		return
	last_events.append({"type": "attack", "from": gizmo_position, "to": target.position})
	target.hp -= attack_damage
	last_events.append({"type": "hit", "position": target.position, "damage": attack_damage})
	if target.hp <= 0.0:
		enemies.erase(target)
		var spark := Pickup.new()
		spark.position = target.position
		spark.value = target.xp_value
		pickups.append(spark)
		last_events.append({"type": "defeat", "position": spark.position, "xp": spark.value})

func _update_pickups(gizmo_position: Vector3) -> void:
	var kept: Array[Pickup] = []
	for p in pickups:
		if gizmo_position.distance_to(p.position) <= pickup_radius:
			last_events.append({"type": "pickup", "value": p.value})
			if add_xp(p.value):
				last_events.append({"type": "levelup", "level": level})
		else:
			kept.append(p)
	if kept.size() > max_pickups:                    # cap uncollected Sparks
		kept = kept.slice(kept.size() - max_pickups)  # drop the oldest (simulation.ts:945)
	pickups = kept

The weapon (updateWeapons + dealDamage, simulation.ts:817-836, 1169-1185) finds the nearest live enemy in range and deals damage; a kill removes the enemy and drops a Spark worth its xp_value (nibbler = 3, ENEMY_SPECS). _update_pickups (the xp gain, simulation.ts:949, 1038) vacuums Sparks within pickup_radius straight into add_xp — the 0006 leveling curve. That single call is the loop snapping shut.

Two seams for what's next — events & a cap

The combat code also fills last_events, a list rebuilt each tick() (a minimal echo of the GameEvent[] that simulation.ts returns every frame): an attack, a hit, a defeat on a kill, a pickup on collection, a levelup when the curve fires. Nothing reads it yet — it's the hook the 0011 HUD (a level-up flash) and a later VFX lesson will consume, so they never have to diff state to guess what happened. And uncollected Sparks are capped at max_pickups (90, per simulation.ts's MAX_PICKUPS; balance §6 tracks pickup count): over the cap the oldest are dropped, so a wave (0010) of kills can't grow an unbounded pile of pickups. Both are tested.

Test-first — damage, death, drop, collect, level

Each link in the chain gets a check (add to the runner; the enemy-only tests set a huge attack_cooldown so the weapon doesn't interfere):

func _test_autofire_damages_enemy() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 9999.0   # no auto-spawn; we place the enemy
	sim.attack_cooldown = 0.05    # fire on the first tick
	var e := Sim.Enemy.new()
	e.position = Vector3(2.0, 0.0, 0.0)  # within attack_range (6)
	e.hp = 5.0
	e.radius = 1.0
	sim.enemies.append(e)
	sim.tick(0.05, Vector3.ZERO)
	_check("auto-fire damages the nearest enemy", e.hp < 5.0)

func _test_enemy_dies_and_drops_spark() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 9999.0
	sim.attack_cooldown = 0.05
	sim.pickup_radius = 0.0       # isolate: the dropped Spark must not be auto-collected
	var e := Sim.Enemy.new()
	e.position = Vector3(2.0, 0.0, 0.0)
	e.hp = 1.0            # one shot kills (attack_damage 1)
	e.radius = 1.0
	e.xp_value = Sim.NIBBLER_XP
	sim.enemies.append(e)
	sim.tick(0.05, Vector3.ZERO)
	_check_eq("a killed enemy is removed", sim.enemies.size(), 0)
	_check_eq("death drops one Spark", sim.pickups.size(), 1)
	_check_eq("the Spark is worth the enemy's xp", sim.pickups[0].value, Sim.NIBBLER_XP)

func _test_collecting_spark_grants_xp() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 9999.0
	var p := Sim.Pickup.new()
	p.position = Vector3(0.5, 0.0, 0.0)  # within pickup_radius (1.8)
	p.value = 10
	sim.pickups.append(p)
	var xp_before := sim.xp
	sim.tick(0.05, Vector3.ZERO)
	_check_eq("collecting a Spark banks its value", sim.xp, xp_before + 10)
	_check_eq("the collected Spark is removed", sim.pickups.size(), 0)

func _test_sparks_drive_leveling() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 9999.0
	for i in 10:                     # 10 Sparks * 10 = 100 xp > level-1 threshold (92)
		var p := Sim.Pickup.new()
		p.position = Vector3.ZERO
		p.value = 10
		sim.pickups.append(p)
	sim.tick(0.05, Vector3.ZERO)
	_check_eq("collecting enough Sparks levels Gizmo up", sim.level, 2)

The last test is the whole point: drop enough Sparks, collect them, and the 0006 leveling curve fires — the systems from three lessons ago and now are one loop. Review-hardening checks join them (the weapon skips a corpse, picks the nearest of two, the pickup cap holds, and each event fires), so the end state is PASS — 63 checks. Part 2 — render kills & Sparks (the scene)

Now that enemies die and Sparks appear and vanish, the controller's old index-parallel mirror won't do — we need add and remove. Switch to one map per agent type (the refactor 0008 promised). Add a godot/scenes/spark.tscn (a small violet emissive sphere — Sparks are violet, per the art direction), then rewrite the bridge:

const EnemyScene := preload("res://scenes/enemy.tscn")
const SparkScene := preload("res://scenes/spark.tscn")

@export var gizmo: Node3D

var sim := Simulation.new()
var _enemy_views: Dictionary = {}  # Simulation.Enemy  -> Node3D
var _spark_views: Dictionary = {}  # Simulation.Pickup -> Node3D

func _physics_process(delta: float) -> void:
	var gizmo_position := gizmo.global_position if gizmo != null else Vector3.ZERO
	sim.tick(delta, gizmo_position)
	_sync(sim.enemies, _enemy_views, EnemyScene)
	_sync(sim.pickups, _spark_views, SparkScene)

## Mirror data agents onto visuals: add a view for each new agent, move existing
## views, and free the view of any agent that's gone (died / collected).
func _sync(agents: Array, views: Dictionary, scene: PackedScene) -> void:
	for agent in agents:
		var view: Node3D = views.get(agent)
		if view == null:
			view = scene.instantiate()
			add_child(view)
			views[agent] = view
		view.global_position = agent.position
	for agent in views.keys():
		if not agents.has(agent):
			views[agent].queue_free()
			views.erase(agent)

One generic _sync drives both enemies and Sparks: the dictionary is keyed by the data agent itself, so when Simulation removes an enemy (death) or a Spark (collected), the matching visual is freed automatically — no index bookkeeping. Press Play: stand still and a halo of violet Sparks rings you (kills at the weapon's edge); move and you vacuum them, and the Simulation's XP rises (there's no HUD yet — that lands in 0011).

Deliberately simple, for now

The shot is invisible (no projectile/tracer) — weapon VFX is a later particles/shader lesson; here the feedback is enemies vanishing + Sparks. One weapon, one target per shot, no crits — multi-target, crits, Cache/Heart drops, and magnet upgrades come with the upgrade draft. Spark value is the faithful 3, so leveling takes many kills — that's real horde pacing (waves ramp it in 0010).
