Gizmo · Lesson 0019 · Path A refactor

Wire the Beacon into the live run — the first Path A win you can play

0017–0018 proved the rekindle channel and made it the win, all headless. Now it crosses into the running game: the controller authors the real Beacon from the scene, the HUD drops its countdown for a rekindle readout, and the end-screen reads "Beacon Rekindled." Walk Gizmo north, hold the light — and win.

Why this is the bridge, not new mechanics

The rules are done. What's missing is the seam where the headless Simulation meets the scene (ADR 0002): the sim has a beacon at (0,0,0) with radius 0 — inert — and nothing ever places a real one. 0019 is four small wirings that turn "the mechanic is proven" into "you can play it." No new game logic; just connecting the proven engine to the live world.

Part 1 — the controller authors the Beacon

The scene already holds the truth: a bare NorthBeaconDaisZone Marker3D at z = -13, with the glowing placeholder cylinder on top. The controller reads that position and feeds it to the sim — the same seam that mirrors arena obstacles (one source of truth, fed into the rules engine):

@export var beacon_radius: float = 3.0   // controller-side: the bare marker has no radius to read

func _register_beacon_under(root):
    var dais = root.find_child("NorthBeaconDaisZone", true, false)
    if dais is Node3D:
        sim.beacon_position = dais.global_position
        sim.beacon_radius   = beacon_radius      // no marker → stays inert (headless tests, bare scenes)

The radius lives on the controller, not the marker. The markers are plain Marker3Ds — adding a radius field to them would mean editing the off-limits art-stream scene and inventing metadata. The controller is the honest home for it.

Part 2 — the HUD drops the countdown

The old HUD showed 4:00 ticking down — exactly the round-counter the project ruled out (ADR 0003/0005: no player-facing timer). We replace that one label with a Beacon readout, a pure formatter so it's unit-tested:

rekindle_readout(Dormant,  _)   -> "REKINDLE BEACON"     // the objective
rekindle_readout(Rekindling, p) -> "REKINDLING 60%"      // the live channel
rekindle_readout(Rekindled, _)  -> "BEACON REKINDLED"    // the win

(The pure format_clock stays — the end-screen still uses it for a "survived" run-length stat — but the HUD no longer counts anything down.)

Part 3 — a faithful forced win, and the new end-screen

Two smaller wirings finish it. The debug F9 shortcut used to fake a win by spending the clock (elapsed = run_duration) — meaningless now, so it rekindles instead (beacon_state = Rekindled). And the end-screen copy becomes Path A:

PHASE_COMPLETE -> "BEACON REKINDLED"  · "The hearth catches; the cold world holds back."
PHASE_GAMEOVER -> "GIZMO OFFLINE"     · "Gizmo's light failed."
Verified live, not just headless

The 165-check gate proves the logic, but the renamed %ObjectiveLabel and the real-scene marker lookup only count when the game runs. So we ran it: the HUD reads REKINDLE BEACON (no clock); a script confirms the sim received beacon_radius 3 at (0,0,-13) from main.tscn; and holding the radius drives REKINDLING % up to a PHASE_COMPLETE with the end-screen banner BEACON REKINDLED. The win you proved in tests is the win you see on screen.

Done means…

Done = a played run can be won the Path A way — walk to the Beacon, hold its radius, watch the readout climb, and reach the "Beacon Rekindled" screen; HP-0 still loses.
