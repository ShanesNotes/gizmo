The run timer & health

Give the run a clock you can win and an HP bar you can lose — still headless, still test-first.

Why this, now

0006 gave Gizmo progression; now the run needs stakes: a way to win (survive the bounded run) and a way to lose (run out of health). Both already live in simulation.ts — the run clock and the player's HP — so we keep porting, headless and test-first, growing the same Simulation. Same truth hierarchy as before: tuning intent in reference/game-balance-reference.md §5, meaning in design-handoff/NARRATIVE.md, mechanics in simulation.ts.

HP is not the Spark of Humanity (ADR 0001)

This lesson ports the player's HP — the health bar (`hp`/`maxHp`); taking damage lowers it, zero ends the run. That is not the Spark of Humanity meter (a separate objective/survival meter Gizmo keeps alive — "Keep it safe. Keep it alive.") whose mechanics we haven't designed yet. Three distinct things — HP, Sparks, Spark of Humanity — per docs/adr/0001-sparks-hp-spark-of-humanity-are-distinct.md. We build HP here; the Spark of Humanity is deliberately deferred.

The slice: a clock and a health bar

Two pure mechanics from simulation.ts:

  • The run clock. Time advances each frame; the run is bounded at RUN_DURATION = 240s (simulation.ts:202). When it elapses, you win (phase = "complete", simulation.ts:489-493). run_progress and time_remaining mirror lines 533 and 535. Note the dt clamp (min(dt, 0.05), line 463): a lag spike must not fast-forward the whole run.
  • Health. A hit lowers HP, grants brief i-frames (1.58s, so one touch isn't instant death), and floors at 0; reaching 0 is a lose (phase = "gameover", simulation.ts:722-738).

A small phase field ties it together: playing → either complete (won) or gameover (lost). The clock only ticks while playing (simulation.ts:460).

(The enemy-contact geometry and the "no damage for the first 7s" grace belong with enemies (0008); here take_damage is the pure damage primitive.)

func _test_run_completes() -> void:

func _test_damage_and_iframes() -> void:

	_check("first hit lands", sim.take_damage(2))
	_check_eq("hp drops by the damage", sim.hp, 5)
	_check("i-frames block the next hit", not sim.take_damage(2) and sim.hp == 5)

func _test_death_is_gameover() -> void:

Step 2 (GREEN) — the real run clock & health

Replace the stubs:

## Advance the run by dt seconds. Mirrors the top of updateGameState
## (simulation.ts:459-494): no-op unless playing; dt clamped; the run COMPLETES
## (a win) when the timer runs out.
func tick(dt: float) -> void:
	if phase != PHASE_PLAYING:
		return
	var safe_dt := minf(dt, MAX_DT)
	elapsed += safe_dt
	invulnerable = maxf(0.0, invulnerable - safe_dt)
	if elapsed >= run_duration:
		phase = PHASE_COMPLETE

func run_progress() -> float:           # simulation.ts:533
	return clampf(elapsed / run_duration, 0.0, 1.0)

func time_remaining() -> float:         # simulation.ts:535
	return maxf(0.0, run_duration - elapsed)

## A hit lowers HP, grants i-frames, floors at 0; 0 HP ends the run (lose).
## Mirrors simulation.ts:722-738 (minus enemy geometry). HP, not Spark of
## Humanity (ADR 0001). Returns true if the hit landed.
func take_damage(amount: int) -> bool:
	if phase != PHASE_PLAYING or invulnerable > 0.0:
		return false
	hp = maxi(0, hp - maxi(0, amount))
	invulnerable = HIT_INVULN
	if hp <= 0:
		phase = PHASE_GAMEOVER
	return true

Why it's shaped this way

  • A tiny phase machine. playing → complete | gameover is the whole run lifecycle. Win and lose are just two phase transitions; the HUD and screens (0011/0012) will read phase.
  • The dt clamp is a real rule, so we test it. Without min(dt, 0.05), a one-second hitch would jump the clock a full second (or end the run). Faithful to the source, and pinned by a test.
  • i-frames make HP fair. 1.58s of invulnerability after a hit means brushing an enemy costs one tick of HP, not all of it — which is why we tick time forward in the test to prove the window clears.
  • Still RefCounted, still headless. A whole run can be simulated and checked with no window — the same testability that made 0006 fast.
