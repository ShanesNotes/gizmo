The Tick

Give the simulation a heartbeat: one function that advances the world by a slice of time — safely, so a lag spike can never break the game.

0005 ✓ → 0006 · the tick → 0007 · state on screen

The one idea

So far the game state is frozen at t = 0. A game needs to move forward in time. That's update_state(state, input, dt): it takes the current state and dt (the seconds elapsed since the last tick), advances everything by that slice, and returns a list of events that happened (like "the run ended").

The crucial trick is the dt clamp. Normally dt is tiny (~0.016s at 60fps). But if the game stalls — you alt-tab, the OS hiccups — the next dt could be half a second. Simulate that in one jump and the player teleports, collisions tunnel through walls, the run desyncs. So we cap each tick at MAX_TICK_DT (0.05s). If the machine lags, Gizmo runs a hair slow instead of breaking.

Straight from the seed. This mirrors updateGameState()'s Math.min(dt, 0.05) in game-src-phaser/src/game/simulation.ts, with one defensive Godot-side improvement: negative editor/test values clamp to zero instead of rewinding time.

Do it — part 1: the tick function

Open scripts/simulation.gd. Add the clamp constant beside your other constants:

add near the top, with the other const lines
const MAX_TICK_DT: float = 0.05

Then add this function at the bottom of the file:

append to the end of simulation.gd
func update_state(state: Dictionary, input: Dictionary = {}, dt: float = 0.0) -> Array:
    if state.get("phase") != "playing":
        return []

    var events: Array = []
    var safe_dt: float = clampf(dt, 0.0, MAX_TICK_DT)
    state["elapsed"] = float(state["elapsed"]) + safe_dt

    if float(state["elapsed"]) >= float(state["run_duration"]):
        state["phase"] = "complete"
        events.append({"type": "complete"})

    return events

What each piece does

input := {}, dt := 0.0Default arguments — you can call update_state(state) and they fill in. if phase != "playing": return []A finished or paused run doesn't advance. No work, no events. clampf(dt, 0.0, MAX_TICK_DT)The clamp. safe_dt is never negative and never more than 0.05. state["elapsed"] = … + safe_dtTime moves forward. Dictionaries are references — this edits the caller's state directly. events.append({"type": "complete"})When the 240s run ends, flip to complete and report it. Events are how the sim tells the game "something happened."

Do it — part 2: prove time moves

Open tests/run_simulation_tests.gd. Just before the if _failures == 0: line, add a tick and three checks:

add inside _init(), before the pass/fail report
var events := sim.update_state(state, {}, 0.25)
_assert_approx(state["elapsed"], 0.05, "a big dt is clamped to one safe step")
_assert_equal(state["phase"], "playing", "one tick keeps the run playing")
_assert_equal(events.size(), 0, "an opening tick emits no events")

var rewind_events := sim.update_state(state, {}, -1.0)
_assert_approx(state["elapsed"], 0.05, "negative dt does not rewind time")
_assert_equal(rewind_events.size(), 0, "negative dt emits no events")

We pass a deliberately huge dt of 0.25 and a deliberately invalid negative dt. The first advances only 0.05; the second does not rewind time. Run the tests:

godot --headless --path godot --script res://tests/run_simulation_tests.gd

Simulation tests passed — now including the rules about time.

That's the win. The simulation has a heartbeat. update_state is the engine of the whole game — every future system (movement, spawns, XP, the four economies) hangs off this one function, and the dt clamp keeps it stable no matter how the machine behaves.
Make it bite: change the clamp expectation from 0.05 to 0.25 and rerun — you'll see a precise FAIL proving the clamp really is holding the line. Change it back to green.

Mental model to keep

  • update_state(state, input, dt) = advance the world one slice of time, return what happened.
  • Always clamp dt so a stall can't fast-forward the sim into chaos.
  • Events are the sim's outbox — it returns a list of happenings; the game reacts to them later.
  • Dictionaries pass by reference — update_state edits the state you hand it, in place.
