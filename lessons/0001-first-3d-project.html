Your first 3D project

From an empty folder to a lit 3D world you can press Play on.

Why this, first

Every game we will ever build starts as a Godot project — a folder Godot recognizes — and an opening scene it runs when you press Play. Before Gizmo can walk, before a single enemy spawns, we need those two things, set up correctly for 3D. That is the entire win for this lesson: an empty folder becomes a 3D world that runs.

The one idea: a project is just a folder with a project.godot

Godot doesn't have a database or a hidden install. A project is a directory containing a plain-text file named project.godot. Open that folder in Godot and you have a project; that file holds your settings. Ours lives at godot/project.godot, and it's already written. The lines that matter:

[application]
config/name="Gizmo"
config/features=PackedStringArray("4.6", "Forward Plus")

[rendering]
renderer/rendering_method="forward_plus"
The decision that makes it 3D

The old shell we cleared out used the GL Compatibility renderer — a lightweight path built for 2D and old hardware. It can show 3D, but it can't give us the lighting and materials a painted, gouache look needs. We switched to Forward+: Godot's full desktop renderer (it runs on Vulkan, which your machine has). This single line is the difference between a flat 3D and one that can hold real light. Rule of thumb: 3D desktop game → Forward+.

A scene is a tree of nodes

If the project is the folder, the scene is what's actually in the world. A scene is a tree: one root node with children. A node is one small capable thing — "a camera," "a light," "a chunk of mesh." You compose them into a tree, and that tree is your level. For our first 3D scene we need exactly four kinds:

Main (Node3D) the root — an empty 3D anchor ├─ Camera3D (Camera3D) the eye that renders the view ├─ Sun (DirectionalLight3D) light from a direction, like a sun ├─ Ground (MeshInstance3D) a flat PlaneMesh to stand on └─ Box (MeshInstance3D) a placeholder BoxMesh (Gizmo's stand-in)

A Node3D is the simplest 3D node: it has a position, rotation, and scale, and nothing else. It makes a good root — a single handle for the whole scene. A MeshInstance3D is a node that draws a mesh; we'll give one a flat plane (the floor) and one a box (a cheap stand-in for Gizmo until lesson 0003).

Build the slice — your call how

Two ways to build this scene. Pick whichever you like for this slice; you can switch any time, lesson to lesson.

A) Hands-on in the editor (the default — doing it once builds intuition):

  1. Open Godot, click Import, choose godot/project.godot, open it.
  2. In the Scene dock (top-left): Other Node → search Node3D → Create. Double-click it and rename it Main.
  3. Select Main, click the + (Add Child Node), add a Camera3D. Then add a DirectionalLight3D (rename it Sun), and two MeshInstance3D nodes (rename Ground and Box).
  4. Select Ground. In the Inspector (right), find Mesh → New PlaneMesh. Click the PlaneMesh and set its size to about 20 × 20.
  5. Select Box: Mesh → New BoxMesh. In its Transform → Position, set Y to 0.5 so it sits on the floor.
  6. Position the camera so it looks down at the box: select Camera3D, set Position to about (0, 6, 8) and Rotation X to -35°. (We'll make this the proper Diablo angle in lesson 0002 — eyeballing is fine now.)
  7. Ctrl+S → save as scenes/main.tscn. When Godot asks to set a main scene (or via Project → Project Settings → Run), choose main.tscn.
  8. Press F5 (Play). A window opens onto a lit floor with a box on it. 🎉

B) Hand it to me. Say the word and I'll build this exact tree through the Godot runtime, then walk you through every node I added and why — same result, nothing left as a black box.

How you'll know it worked

Press Play and you should see a window with a flat ground plane, a box resting on it, and visible shading from the light (one side of the box brighter than the other — that's Forward+ doing its job). If the window is black, the camera isn't pointed at anything: check its position and that its rotation tilts it downward. Stuck? Ask me — that's what I'm here for.
