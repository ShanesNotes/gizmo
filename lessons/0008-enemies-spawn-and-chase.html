Gizmo · Lesson 0008 · The loop

Enemies: the first pressure machine

Spawn a crowd that chases Gizmo and hurts on contact — the first thing in the game that pushes back.

Why this, now

You can move, the run can be won or lost — but nothing threatens Gizmo yet. Enemies are the run's first pressure machine: they force kiting decisions, they'll drop Sparks and trigger the level-up dopamine later, and they're the foundation for the pressure director. For the horde loop we're after (Vampire Survivors / Diablo / Megabonk), enemies must be balance data first — and the crowd must read as a crowd. Tuning intent: reference/game-balance-reference.md §3 (defense), §5 (scaling/horde), §6 (caps); mechanics: simulation.ts (ENEMY_SPECS, updateEnemies).

The architecture decision (ADR 0002)

This is where the tested Simulation finally meets the running game. We chose a hybrid: Simulation owns the rules (enemies live there as data — spawn, chase, separation, contact, HP), and the scene renders them. A GameController node bridges: each physics frame it feeds Gizmo's position into Simulation.tick(dt, gizmo_position), then mirrors each enemy onto a visual. Gizmo's own movement stays in Godot (0004–0005). And we use simple seek + soft separation, not NavigationAgent3D and not physics bodies — many cheap data agents are clearer and testable. Crucially, crowd readability is solved in Simulation (the positions it mirrors don't overlap), not with scene-only collision. Full rationale: docs/adr/0002-simulation-owns-rules-scene-renders.md.

How this maps to the balance reference
  • Spawn cadence (spawn_interval) → §5.2 (time is the scaling axis — "the clock is the boss") + §5.3 (horde tuning). Flat for now; ramping pressure is 0010.
  • Alive-enemy cap (max_enemies) → §6.1 (cap + spawn pressure). A safety bound now; the pressure director tunes it later. (simulation.ts caps at 122.)
  • Contact + i-frames + 7s opening grace → reuses 0007's take_damage (§3.4), gated by a grace (simulation.ts:722) so a fresh run isn't lethal on the first spawn.
  • Separation → soft push-apart so the pack reads as a crowd (defensive readability; §3/§6), cheaper and more testable than physics bodies.
  • Enemy stats (the nibbler) → ENEMY_SPECS (simulation.ts:259); relative balance kept (damage 1), positions/speeds in Godot metres (ADR 0002).

Part 1 — enemies as data agents (logic, test-first)

An enemy is just a little bag of numbers. In godot/scripts/simulation.gd, an inner class:

class Enemy extends RefCounted:
	var position := Vector3.ZERO
	var hp: float = 1.0
	var speed: float = 0.0
	var radius: float = 1.0
	var damage: int = 1
	var kind: String = ""

tick grows a second argument — where Gizmo is — and runs the enemies in four phases: spawn (capped) → seek → separate → contact. (The default keeps the 0006/0007 tests, which call tick(dt), working.)

func tick(dt: float, gizmo_position := Vector3.ZERO) -> void:
	if phase != PHASE_PLAYING:
		return
	var safe_dt := clampf(dt, 0.0, MAX_DT)
	elapsed += safe_dt
	invulnerable = maxf(0.0, invulnerable - safe_dt)
	_update_enemies(safe_dt, gizmo_position)
	if elapsed >= run_duration:
		phase = PHASE_COMPLETE

func _update_enemies(dt: float, gizmo_position: Vector3) -> void:
	# 1. Spawn on a cadence, up to the alive cap (balance §6.1).
	_spawn_timer += dt
	if _spawn_timer >= spawn_interval:
		_spawn_timer -= spawn_interval
		if enemies.size() < max_enemies:
			_spawn_nibbler(gizmo_position)
	# 2. Seek toward Gizmo on the floor plane.
	for e in enemies:
		var to_player := gizmo_position - e.position
		to_player.y = 0.0
		var dist := to_player.length()
		if dist > 0.0001:
			e.position += to_player / dist * e.speed * dt
	# 3. Soft separation so overlapping enemies read as a crowd, not one blob.
	_separate_enemies()
	# 4. Contact — AFTER moving, within the player's reach, past the opening grace.
	if elapsed > CONTACT_GRACE:
		for e in enemies:
			var to_player := gizmo_position - e.position
			to_player.y = 0.0
			if to_player.length() <= e.radius + PLAYER_CONTACT_RADIUS:
				take_damage(e.damage)  # reuses the 0007 primitive (i-frames!)

func _separate_enemies() -> void:
	var count := enemies.size()
	for i in count:
		for j in range(i + 1, count):
			var a := enemies[i]
			var b := enemies[j]
			var delta := b.position - a.position
			delta.y = 0.0
			var d := delta.length()
			var min_d := a.radius + b.radius
			if d < min_d:
				var push: Vector3
				if d > 0.0001:
					push = delta / d * (min_d - d) * 0.5
				else:
					push = Vector3(cos(i + j), 0.0, sin(i + j)) * min_d * 0.5
				a.position -= push
				b.position += push

The seek is the whole "AI": a unit vector toward Gizmo × speed × dt. Separation pushes overlapping enemies apart by half their overlap each, so a pack reads as many bodies (O(n²) is fine for a teaching slice — the alive cap, and a spatial hash later, keep it bounded). Contact is checked after moving, within radius + PLAYER_CONTACT_RADIUS, and only past the 7-second grace (simulation.ts:722) — then it just calls take_damage, so i-frames and the lose condition from 0007 come for free. _spawn_nibbler drops enemies on a ring inside the floor (radius 9 on the 20×20 ground), at a deterministic golden-angle spread.

Test-first — spawn, chase, separate, contact (with the grace)

The tests grow with the behavior (add to the runner, call from _initialize()):

func _test_enemy_spawns_on_cadence() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.2
	_check_eq("no enemies at the start", sim.enemies.size(), 0)
	for i in 4:
		sim.tick(0.05, Vector3.ZERO)  # 0.20s -> one spawn
	_check_eq("one enemy after one interval", sim.enemies.size(), 1)

func _test_enemy_chases_gizmo() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.05
	var target := Vector3(5.0, 0.0, 0.0)
	sim.tick(0.05, target)
	var e := sim.enemies[0]
	var before := e.position.distance_to(target)
	sim.tick(0.05, target)
	_check("enemy moves toward Gizmo", e.position.distance_to(target) < before)

func _test_enemy_contact_damages_player() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.05
	sim.elapsed = 8.0  # past the 7s contact grace (simulation.ts:722)
	sim.tick(0.05, Vector3.ZERO)
	var e := sim.enemies[0]
	var hp_before := sim.hp
	sim.tick(0.05, e.position)  # Gizmo on the enemy -> contact
	_check_eq("contact costs the player 1 hp", sim.hp, hp_before - 1)
	sim.tick(0.05, e.position)  # again immediately -> blocked by i-frames
	_check_eq("contact respects i-frames", sim.hp, hp_before - 1)

func _test_contact_grace() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.05
	sim.tick(0.05, Vector3.ZERO)  # before the grace; spawn an enemy
	var e := sim.enemies[0]
	sim.tick(0.05, e.position)  # on the enemy, but before 7s -> no damage
	_check_eq("no contact damage before the 7s grace", sim.hp, 7)
	sim.elapsed = 8.0
	sim.tick(0.05, e.position)  # past the grace -> contact lands
	_check("contact damages after the grace", sim.hp < 7)

func _test_enemies_separate() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.05
	sim.tick(0.05, Vector3.ZERO)
	sim.tick(0.05, Vector3.ZERO)  # two enemies
	sim.enemies[0].position = Vector3.ZERO
	sim.enemies[1].position = Vector3(0.3, 0.0, 0.0)  # overlapping (radii sum to 2.0)
	var before := sim.enemies[0].position.distance_to(sim.enemies[1].position)
	sim._separate_enemies()
	_check("overlapping enemies push apart", sim.enemies[0].position.distance_to(sim.enemies[1].position) > before)

func _test_enemy_cap() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.05
	sim.max_enemies = 3
	for i in 60:
		sim.tick(0.05, Vector3.ZERO)
	_check_eq("spawning stops at the cap", sim.enemies.size(), 3)

func _test_no_spawning_after_run_over() -> void:
	var sim := Sim.new()
	sim.spawn_interval = 0.05
	sim.take_damage(100)  # -> gameover
	for i in 10:
		sim.tick(0.05, Vector3.ZERO)
	_check_eq("a dead player spawns no enemies", sim.enemies.size(), 0)

Implement spawn → cadence/cap checks go green; add the seek → "moves toward Gizmo" greens; add separation → the push-apart check greens; route contact through take_damage with the grace → the damage/i-frame/grace checks green. Part 2 — render the enemies (the scene)

The logic is the truth; the scene just draws it. Two pieces:

  1. A visual — godot/scenes/enemy.tscn: a Node3D root (the agent's ground position) with a small reddish SphereMesh child lifted to sit on the floor. No physics, no script — pure visual; the controller moves it.
  2. The bridge — godot/scripts/game_controller.gd on a GameController node in godot/scenes/main.tscn:
extends Node3D

const EnemyScene := preload("res://scenes/enemy.tscn")

@export var gizmo: Node3D   # assign Gizmo in the Inspector

var sim := Simulation.new()
var _views: Array[Node3D] = []  # one visual per sim enemy, parallel by index

func _physics_process(delta: float) -> void:
	var gizmo_position := gizmo.global_position if gizmo != null else Vector3.ZERO
	sim.tick(delta, gizmo_position)

	while _views.size() < sim.enemies.size():   # a visual for each new enemy
		var view: Node3D = EnemyScene.instantiate()
		add_child(view)
		_views.append(view)

	for i in sim.enemies.size():                 # mirror positions
		_views[i].global_position = sim.enemies[i].position

Add a GameController node to godot/scenes/main.tscn and set its Gizmo export to the Gizmo node. Press Play: red nibblers spawn on the ring and converge into a spaced crowd around Gizmo (separation keeps them legible). Stand still past the grace and they swarm and kill you (gameover); move and you kite — the first real decision the game asks.

Deliberately simple, for now

Enemies don't die yet (no weapons — that's 0009 combat, which also adds Spark drops), and spawn on a flat cadence (ramping pressure is 0010). Separation is O(n²), bounded by the alive cap; a spatial hash comes if pressure pushes counts higher. The index-parallel _views mirror works because enemies only grow in 0008; once they can die, the controller switches to a per-enemy map with removal.

Done means…

Done = the suite is green (43 checks), and in-game red enemies spawn, chase, and form a spaced crowd (not a blob) that hurts Gizmo on contact after the grace (standing still → gameover).
