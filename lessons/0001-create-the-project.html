Create the Project

Open Godot, make a new project, and land in the editor. No files, no code — just hit Create.

0001 · create the project → 0002 · first scene + Play → 0003 · first GDScript

The one idea

Every Godot project begins the same way: you open the engine, click New Project, pick a folder, and hit Create. Godot scaffolds everything for you — including the one marker file, project.godot, that makes the folder a project. You never hand-write that file; the editor owns it.

Godot is editor-first. We build this whole game inside the editor, one visible win at a time — and this first win is the simplest one there is: a real project, open and running.

You don't need to read the seed. Gizmo already exists as a designed web prototype (Phaser/TypeScript) with a full port map. That's your teacher's reference — it lets the AI know exactly what to build next, so we never stall on "what now?" or waste your time. You build in the editor; the teacher leans on the seed. Curious anytime? Just ask.

Do it

  1. Launch Godot 4.6. The Project Manager window opens — a list of projects (empty for now) with buttons on top.
  2. Click + Create (top-left). The New Project dialog appears.
  3. Name it Gizmo. For Project Path, browse into the existing godot/ folder in this repo and select it — the path field should end in /godot. Do not click "Create Folder"; that nests the project a level deep (godot/gizmo/) and our test commands expect it right at godot/.
  4. Renderer: choose Compatibility. It's the lightweight OpenGL path — the right lightweight default for our orthographic 2.5D teaching build, and the version this guide is built against.
  5. Click Create & Edit. Godot builds the project and drops you into the editor.
"The selected path is not empty"? That's just our README.md and a few placeholder folders sitting in godot/ — harmless. There's no project there yet, so Godot will still create one. Continue.
That's the win. You're looking at the Godot editor with an empty project viewport — a real, running project you made the way every Godot developer does. Behind the scenes Godot just wrote godot/project.godot for you. Peek at it if you're curious, but you never have to touch it by hand.

Prove it

The proof is on your screen: the editor is open. If you want a second confirmation, from a terminal at the repo root:

ls godot/project.godot

It exists now — Godot generated it when you hit Create.

Mental model to keep

  • Godot is editor-first. You drive the engine through the editor; it manages the files for you.
  • project.godot = the marker that makes a folder a project. Godot owns it — don't hand-edit it.
  • res:// = your project folder's root. Every path inside the game starts here. You'll meet it in Lesson 0002.
