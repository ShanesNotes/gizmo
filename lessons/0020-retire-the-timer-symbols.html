Gizmo · Lesson 0020 · Path A refactor

Retire the timer symbols — a concept-rename, driven by a failing test

The Beacon is the win now, but the code still talks like a survival timer: run_duration, time_remaining(), run_progress(). Those names are a lie the next reader will believe. We rename the clock to what it actually is — pressure_clock, the director's fuel — and delete the countdown getters entirely. The trick: let a test demand the new world first.

Why a rename is worth a whole lesson

Names are the cheapest documentation and the most expensive lie. After 0016–0019 the clock no longer ends the run — yet run_duration and time_remaining() still sit in the sim, whispering "this is a survival timer" to anyone who reads it. ADR 0005 is explicit: the clock "is renamed in concept to the pressure_clock and survives only as the temporal fuel for the director… never shown to the player as a countdown." Leaving the old names is how a refactor silently rots back toward the game you left.

Part 1 — rename by failing test, not find-replace

A blind sed run_duration → pressure_clock would work and teach nothing — and worse, it would happily keep the countdown getters alive. So we drive it the TDD way: rewrite the test to assert the world we want, watch it go red, then make it true.

func _test_run_clock_is_retired_to_pressure_fuel():
    var sim = Sim.new()
    _check("no player-facing run_progress getter",   not sim.has_method("run_progress"))
    _check("no player-facing time_remaining countdown", not sim.has_method("time_remaining"))
    sim.pressure_clock = 1.0
    for i in 5: sim.tick(0.05)              // elapsed 0.25 of a 1.0 horizon
    _check("pressure_clock is the ramp horizon", sim.pressure() > 0 and sim.pressure() < 1)

Red on two counts: the getters still exist (has_method is true), and pressure_clock doesn't (assignment errors). The test now describes the destination.

Part 2 — make it true

Then, and only then, the edits — and the failing test tells you when you're done:

const RUN_DURATION  -> const PRESSURE_CLOCK        // the pressure-ramp horizon
var   run_duration  -> var   pressure_clock
func run_progress() / func time_remaining()        // DELETED — countdown getters, gone
func pressure(): elapsed / pressure_clock          // the one real consumer, renamed

elapsed stays — it's genuine fuel (accumulated sim time the pressure ramp reads), and ADR 0005 keeps it as "a debug/tuning value." We retire the countdown, not the clock's tick.

The last player-path reference — the end-screen stat

One countdown reference hid on a player path: the end-screen's "SURVIVED" stat clamped elapsed to run_duration. With the clock no longer bounding the run, that clamp is meaningless — so it now reads raw elapsed. The result is that pressure_clock is touched by nothing the player sees: it's pure director internals, exactly as ADR 0005 intends.

Done means…

Done = no run_duration / time_remaining / run_progress anywhere but in comments that note they're gone; pressure_clock is the director's horizon and the player never sees a countdown. Verify: full gate 166 (sim 89 · balance 43 · controller 16 · hud 12 · end-screen 6), and a grep for the old names returns only "retired" comments. You can explain why a rename earns a lesson, why we drove it with has_method assertions instead of find-replace, and why elapsed stayed while the getters went.
