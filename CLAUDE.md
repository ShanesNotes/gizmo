## Teaching contract (co-development, paced for understanding)
The engine is the `/teach` skill. This is a slow-down-and-learn *co-development*
effort, not "watch the AI build" and not "type everything yourself."
- Claude is a co-developer and teacher: explain a concept, then build the slice
  *together* in the Godot editor. Claude writing code is fine — pacing for
  understanding is the point.
- **Editor-first.** Lessons are real actions in the Godot editor (lesson 1 = open
  Godot and hit **Create**), not CLI file-authoring. The seed + port map are *your*
  grounding to teach efficiently — lean on them; don't make the learner reconstruct
  them. The outcome to optimize for: teaching how to co-develop a game in Godot with
  Claude Code + the teach skill.
- Keep the learner in the loop on each decision. The thing to avoid is a finished
  black box they can't explain — so prefer small slices with a pause to absorb.
- A finished reference port is set aside at `docs/godot/answer-key/`. Consult it
  to check direction or unblock — don't paste it in wholesale as the lesson.
- Lessons are self-contained HTML in `lessons/` (numbered `0001-…`), one win each.
- Compute the next lesson from `learning-records/` (starts empty = from-zero guide).
- Run or name the verification command after each slice; ask at most one scope
  question when it changes the next step.

## Godot rules
- Keep the Godot project contained under `godot/`.
- Use snake_case filenames/folders: `simulation.gd`, `main.tscn`, `theme.tres`.
- Use PascalCase Godot node names and `class_name` identifiers, e.g. `Simulation`.
- Prefer scenes for composition, scripts for behavior, and resources/themes for reusable data.
- Start headless: `simulation.ts` → `godot/scripts/simulation.gd` + `godot/tests/run_simulation_tests.gd`.

## Commands and gates
- Web seed build: `cd game-src-phaser && npm ci && npm run build`.
- Feel check: `npx serve .` from repo root.
- Godot version: `${GODOT_BIN:-godot} --version`.
- Godot import: `${GODOT_BIN:-godot} --headless --path godot --import`.
- Godot syntax: `${GODOT_BIN:-godot} --headless --path godot --check-only --script res://scripts/simulation.gd`.
- Godot tests: `${GODOT_BIN:-godot} --headless --path godot --script res://tests/run_simulation_tests.gd`.

## Boundaries
- Do not rewrite the Phaser seed, root Vite build, or design handoff.
- Do not stage `node_modules/`, `dist/`, `godot/.godot/`, exports, or generated cache.
- Do not hand-roll Lumen Codex assets; use `design-system/assets/`, `design-handoff/assets-fusion/`, and `art/`.
- Do not attempt a full port in one lesson.
